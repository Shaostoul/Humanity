# UI review (2026-06-21)

First pass using the new headless snapshot tool (`just snapshots` ->
`tests/snapshots/*.png`).

**Update (same day):** the snapshots now render with a populated **demo
`GuiState`** (`demo_state()` in `ui_snapshots.rs`) instead of the empty default,
so pages show realistic loaded content (vitals, places, garden towers, mining,
chat messages, tasks, listings, etc.) - matching what the operator actually sees.
A **warm-up frame** was also added to the renderer (egui Windows/Areas settle
their layout on the second frame), which fixed the previously-blank **main menu**
and any Window/modal-based page. The per-page notes below were written against the
first (empty) pass; the bugs still stand.

## Concrete bugs (fix first, low risk)

1. **Tasks: duplicate ScrollArea id. FIXED (v0.497.0).** The three Kanban columns
   (Todo / In Progress / Done) shared one ScrollArea id (they live inside
   `ui.columns(3,..)`); egui flagged it with the red "First/Second use of
   ScrollArea ID" overlay and the columns could stop scrolling independently.
   Fixed with `.id_salt(*col_name)` per column.
2. **Chat: the "Send" button was clipped. FIXED (v0.497.0).** The input reserved
   only 104px for the four trailing widgets (search / pins / help / Send), so Send
   overflowed the right edge. Now reserves ~210px (`available_width - 210`,
   floored at 120) so all four fit.

## Cross-cutting patterns

3. **Wasted width on wide screens.** Several pages (Inventory, Profile, parts of
   Market) hug the left ~40% and leave the right half empty black. There is no
   shared "max content width + center" or responsive multi-column rule, so wide
   monitors look unfinished. A single page-width helper (center, or wrap into
   columns past a breakpoint) would fix many pages at once.
4. **Dev affordances leak into the user view.** "Dev: stock seeds" (Inventory),
   "Dev: stock all materials" (Crafting), and "Run scripts/build-library.js"
   (Library empty state) are developer tools/messages a normal user should never
   see. Gate them behind a dev flag, and write user-facing empty states.
5. **Empty states are uneven.** Market ("No listings / Create one to get
   started") is good. Others are dev-y or terse. A consistent empty-state pattern
   (one line of what-this-is + one primary action) would lift the whole app.

## Per-page notes

- **Humanity (the landing / Mission Dashboard):** a wall of manifesto prose, no
  visual hierarchy, no scannable cards, no quick actions. It is the first thing a
  new user sees; it should welcome and orient (a few cards: what this is, start
  here, the mission), not read like a dense essay.
- **Chat:** strong. Clean three-column Discord-style layout (connection + DMs +
  groups + servers / channel / studio + friends + members). Just the clipped Send
  button.
- **Crafting:** strong master-detail (categories / recipe list / details pane).
- **Profile:** nice pattern worth spreading: the sidebar groups sections by
  privacy with color-coded dots (PRIVATE red / PERSONAL amber / PUBLIC green).
  Content hugs the left though.
- **Market:** clean; the Categories sidebar is empty (should list categories).
- **Inventory:** clear collapsible sections (Status / Equipment / Backpack /
  Garden / Mining), but left-hugging and full of dev buttons.
- **Library:** dev empty-state only (no data in the test env).
- Not yet deep-reviewed: governance, identity, wallet, quests, calendar, notes,
  cosmos, settings sub-pages (PNGs exist in tests/snapshots/).

## Suggested priority

1. Fix the Tasks ScrollArea id clash + the Chat Send clip (clear bugs).
2. A shared page-width / centering helper, applied to the left-hugging pages.
3. Gate dev affordances behind a dev flag; write real empty states.
4. Redesign the Humanity landing into scannable cards with a "start here".

The snapshot tool makes all of this iterable: change a page, `just snapshots`,
look at the PNG.
