# Design

HumanityOS's design docs (architecture, protocol specs, game systems, UI) live in the
repo's `docs/design/`, `docs/network/`, `docs/game/`, and `docs/contributor/` folders,
not as individually rendered pages on this site. This page previously listed ~150
category links to pages that were never actually built (confirmed 2026-06-30, every
one 404'd), so rather than continue promising pages that don't exist, it now points
straight at the real source:

- **[Browse docs/design/ on GitHub](https://github.com/Shaostoul/Humanity/tree/main/docs/design)**, architecture, storage, UI system, and feature-specific design docs.
- **[Browse docs/network/ on GitHub](https://github.com/Shaostoul/Humanity/tree/main/docs/network)**, the relay protocol, object format, and federation model.
- **[Browse docs/game/ on GitHub](https://github.com/Shaostoul/Humanity/tree/main/docs/game)**, gameplay systems and simulation design.
- **[Browse docs/contributor/ on GitHub](https://github.com/Shaostoul/Humanity/tree/main/docs/contributor)**, the architecture overview and contributor onboarding sequence.
- **[CLAUDE.md](https://github.com/Shaostoul/Humanity/blob/main/CLAUDE.md)**, the single source of truth for the as-built architecture, file map, and non-negotiable design rules.

*Design defines what must be true. Implementations make it real.*
