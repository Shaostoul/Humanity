# Database Concepts

SurrealDB schema definitions (`.surql` files) modeling the full knowledge domain for HumanityOS, organized by category: beings, communications, engineering, entities, humanities, mathematics, morality, resources, science, technology, and users.

## Contents

- `namespace.surql` — Top-level namespace definition
- `db_definitions.surql` — Database and relationship type definitions
- `db_being_t_*.surql` — Being tables (abilities, bodies, health, mind, nutrition, relationships, soul)
- `db_comms_t_*.surql` — Communication tables (messages, conversations, tickets, VoIP, notifications)
- `db_engineering_t_*.surql` — Engineering tables (civil, electrical, mechanical, software, structural)
- `db_entities_t_*.surql` — Entity tables (characters, crops, fleets, livestock, ships, settlements, stellar objects)
- `db_humanities_t_*.surql` — Humanities tables (anthropology, culture, governance, history, language, philosophy)
- `db_mathematics_t_*.surql` — Mathematics tables (algorithms, constants, formulas, geometry, statistics)
- `db_morality_t_*.surql` — Morality tables (actions, agents, principles, sins, virtues)
- `db_resources_t_*.surql` — Resource tables (commodities, energy, materials, items, land use, water)
- `db_science_t_*.surql` — Science tables (astronomy, biology, chemistry, ecology, geology, physics)
- `db_technology_t_*.surql` — Technology tables (AI, automation, computing, energy systems, robotics, sensors)
- `db_users_t_*.surql` — User tables (profiles, sessions, users)
