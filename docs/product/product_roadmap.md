# Product Roadmap (Draft v0.1)

## Guiding Strategy
Ship the smallest secure, useful core first.
Then expand capabilities without breaking privacy or overextending scope.

---

## Phase 0 - Foundation Alignment (Now)
- Finalize vision, architecture, security principles
- Define MVP boundaries
- Establish naming, docs, and repo structure
- Produce initial UX/system flows

**Exit Criteria**
- Signed-off architecture draft
- Signed-off MVP feature list
- Security baseline accepted

---

## Phase 1 - Core Ecosystem MVP
### Goals
- Unified identity
- Basic secure messaging
- Web + native parity for essential features
- Minimal backend footprint

### Deliverables
- Account/device onboarding
- 1:1 and small-group E2EE chat
- Contact discovery baseline
- Relay fallback for poor networks
- Basic profile/presence

**Exit Criteria**
- Stable daily use by internal alpha group
- Clear telemetry on reliability (privacy-preserving)

---

## Phase 2 - Service Layer + Economy Foundations
### Goals
- Introduce marketplace/service discovery primitives
- Enable ethical commerce pathways
- Integrate trust/reputation controls

### Deliverables
- Service listings + categories
- In-app and web discovery interface
- Partner purchase routing
- Reputation signals (non-invasive)

**Exit Criteria**
- First usable service transactions
- Moderation and abuse controls operational

---

## Phase 3 - Project Universe Interface Bridge
### Goals
- Make immersive shell a first-class interface to ecosystem features

### Deliverables
- In-world social access (chat/presence/contacts)
- In-world marketplace kiosks
- Mission/event hooks tied to ecosystem identity
- Shared account continuity across surfaces

**Exit Criteria**
- Users can perform practical ecosystem actions inside game interface

---

## Phase 4 - Expanded Platform Reach
### Goals
- Device and modality expansion

### Deliverables
- Enhanced mobile experience
- Desktop deep features
- Console/XR integration planning and pilots
- Accessibility and localization improvements

**Exit Criteria**
- Multi-surface continuity is smooth and reliable

---

## Phase 5 - Federation/Decentralization Enhancements
### Goals
- Reduce single-point dependency and improve ecosystem resilience

### Deliverables
- Optional federated relay/community node model
- Policy framework for trusted operators
- Self-host pathways for advanced communities

**Exit Criteria**
- Controlled federation pilot successful
