## What shipped today (v0.712 → v0.714, all released + version-synced)

The theme was the operator's priority #1: CHAT "Servers" fully developed for daily use.
Before today, the saved-servers sidebar could ADD a server but the entries were inert
labels — the Add Server modal even PROMISED "switch by clicking its name" and
`ChatServer.url`'s own doc comment described click-to-reconnect, but neither was wired.

- **v0.712.0** — Saved servers are clickable to SWITCH: clicking one points
  `server_url` at it and reconnects with the same identity (mirrors the Connect
  button: `connect_with_kyber`, reset reconnect timers, clear messages, reload
  history). The active server shows in the success color with a "(current)" tag and
  isn't re-clickable. Each non-current server gets a small "x" to FORGET the bookmark.
- **v0.713.0** — Switching resets the active channel to `#general` (every relay seeds
  it) so switching to a server that lacks your previous channel/DM/group never shows
  an empty view. Matches the existing general-fallback used on leave-DM/disband-group.
- **v0.714.0** — Add Server accepts a BARE HOST: typing `server1.example.com` (no
  scheme) is treated as `https://<host>` for validation + saving, instead of silently
  greying out the Add button. The saved-servers surface is now a full loop:
  add → switch → use → forget.

Each Rust minor got its `just build-game` exe stamp (v0.712.1 / v0.713.1 / v0.714.1).
