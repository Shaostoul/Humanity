# 2026-06-30: Reacquaintance assessment + the great docs cleanup (first Sonnet 5 session)

Not a code-shipping session. The operator's first session with Claude Sonnet 5 (after
Opus 4.8, and a short earlier run on Fable 5) opened with a request to reacquaint with
the whole project and assess where it stands. That assessment surfaced enough real
findings that the rest of the session became the largest documentation cleanup this
project has had.

## The reacquaintance assessment

A 13-agent research pass (5 orienting on strategic state/features/ops/architecture/
codebase reality, 8 doing a "what's missing for a holistically complete app" gap
analysis across gameplay, UI parity, governance, data, security, sustainability,
AI-participation, and production ops) produced a full picture: the autonomous bulk-run
loop had paused itself at v0.637 after 9 verified mothership-superstructure releases,
logging three open forks (M1 zone-editor architecture, M3 civic mall design, grid S3
multi-home tiers) still awaiting operator steer as of this session. ROADMAP.md, STATUS.md,
and PAGES.md were all found significantly stale against the actual v0.637 binary.

## Untracked files: OpenClaw resurfaced

Git status showed several untracked root files (`AGENTS.md`, `IDENTITY.md`, `SOUL.md`,
`HEARTBEAT.md`, `USER.md`, `TOOLS.md`, `.openclaw/`). Investigation confirmed this was
a generic OpenClaw personal-assistant template (Discord/WhatsApp heartbeats, gateway
restarts, TTS storytelling), the same set added Feb 2026 ("trying to get local AI
working") and removed March 2026 ("clean root, unified docs restructure"), now
resurfaced. The operator confirmed: delete it again.

## The docs cleanup sweep

A 7-agent parallel pass (docs/design/ in 3 batches of 45, docs/game/, docs/network/,
docs/reference/+admin/+user/, and docs/website/) found the scale of the problem was
much bigger than the multi-crate docs alone. HumanityOS's `docs/` tree turned out to
carry an entire second, unbuilt "alternate universe" of design content, generic
AI-research tangents (an epistemology mechanic, a proposal to turn the chat relay
into a $285k-$410k Bittensor TAO-mining subnet), a SurrealDB-schema reference layer
(**133 `.surql` files**, zero references to SurrealDB anywhere in `src/`, this project
runs SQLite), a fictional network protocol vocabulary ("spaces," session tokens,
Tailscale onboarding, a CBOR frame protocol that was never built, 16 files deleted
from `docs/network/` alone), a dead multi-crate module lineage duplicated into
`docs/game/` (`module-carpentry.md`, `core-lifeform-model.md`, etc., same pattern as
the contributor docs), and OpenClaw references threaded into `docs/design/`,
`docs/network/`, and `docs/reference/` treating a twice-rejected tool as live
architecture.

**The most serious finding**: `docs/website/knowledge/config-policy.md` rendered
`{% include canon/docs/reference/OPENCLAW_CONFIG_CHANGE_POLICY.md %}`, and
`deploy-pages.yml` actually mirrors the whole `docs/` tree into the Jekyll build at
deploy time, meaning this page would have gone live on the public site with the
operator's local Windows filesystem paths (`C:\Users\Shaos\.openclaw\openclaw.json`)
and an internal AI-tool config policy. Deleted. A systematic follow-up check of every
`{% include canon/... %}` target on the site found 5 more dead includes (3 more
OpenClaw pages under `/reference/`, the orphaned `/design/decisions/adr-0001` page,
and `/knowledge/operations`, whose source doc no longer exists and which arguably
shouldn't have been public either way) and one page, `docs/website/design.md`, whose
~150 category links turned out to all be dead, not from anything this session
touched, there were never any wrapper pages behind any of them. Replaced with a short
page pointing at the real `docs/` folders on GitHub instead of promising 150 pages
that don't exist.

Net: roughly 180 dead/duplicate files removed or archived to `docs/history/` across
the whole tree, several real design docs corrected in place (identity.md's Ed25519 to
Dilithium3 correction, object_format.md's signature algorithm fix, server_federation.md
and social_graph.md's crypto corrections, platform-architecture.md's Tauri to native-egui
rewrite), and 0 broken links confirmed via `scripts/check-doc-links.js` after all of it.

## Direction reset + mothership zone population (same day, second half)

The operator corrected course on two fronts and gave real build direction:

- **No game/simulator toggle.** Rejected as "too confusing from the start." The real
  direction: multiple saves, each house/character flagged real or fake. Real means it
  maps to actual life (the operator's real house, eventually real resources: clothing,
  car, furniture, entered as data). Default app state is real-life; sim/game loads
  only for a fake save or an explicit Settings opt-in, not unconditionally like today's
  early-dev Esc-from-chat shortcut. Nothing built yet (confirmed by grep: no real/fake
  flag exists anywhere in the save system), logged as PRIORITIES.md TIER 2 item 9.
- **GUI philosophy captured**: a page only justifies its existence if merging it into
  another would lose something real, added to `docs/design/ui-system.md`.
- **Jekyll retirement, pushed back on timing.** The operator felt the Jekyll site
  (shaostoul.github.io/Humanity) could be retired now that docs are servable through
  the real website and the app. Investigation found half of that premise was actually
  false: `data/library/` (what the native Library page reads, `docs/` doesn't ship
  with the exe) had drifted from `docs/accord/` for months because the existing resync
  script, `scripts/build-library.js`, just hadn't been re-run. Fixed and documented in
  CLAUDE.md next to the matching theme.ron rule. The real website (`web/`) still has
  zero doc-browsing pages and the deploy pipeline doesn't rsync doc content to the
  nginx web root at all, so Jekyll was NOT retired this session, that would have been
  a real regression. Recommended building the missing web-side piece first.
- **Mothership build-out priority, confirmed**: the player's own home first, it has to
  work before anything mirrors from it, then mothership population.
