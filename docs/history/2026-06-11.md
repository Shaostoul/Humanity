**v0.416.0 — retired-page cleanup + THE RELAY-BUILD FIX.**
- The find of the session: `save_load` (added v0.381) imports the native-gated
  `persistence` module but was itself ungated, so every relay-feature build —
  the CI Deploy-to-VPS workflow AND `just sync` — had failed for **25
  consecutive releases**, and the live relay binary never rebuilt. Nobody had
  looked at CI after pushing. One-line `#[cfg(feature = "native")]` gate fixed
  it; the v0.416.0 deploy run went **green** (first since v0.380) and the VPS
  relay restarted healthy. New CLAUDE.md gotcha: always
  `cargo check --features relay --no-default-features` before pushing Rust
  (the repo's `just install-hooks` pre-commit runs exactly this — operator to
  install once; the classifier blocks an AI doing it).
- The deferred dead-renderer one-pass: `pages/play.rs` and `pages/resources.rs`
  deleted (with their GuiPage variants + plumbing); the standalone onboarding
  page renderer deleted (quest-chain machinery kept). First boot now lands on
  the **Humanity Mission Dashboard**; the boot-page picker offers Humanity +
  Library; legacy saved "onboarding"/"resources" boot strings migrate.
- Profile's Quests section retired: live sim quests now render on the top-level
  **Quests** page above the learn-by-doing chains — the operator's
  "one page, two kinds" quest model finally on one page.
