# 2026-06-21: Native voice arc (v0.485 → v0.495) - mic → DSP → WebRTC → live audio

The arc that gave the **native** client a full voice stack, from a local mic-loopback
test up to live two-way Opus audio that is audible to/from the web client. Everything
is **pure Rust, no C toolchain** (cpal + unsafe-libopus + nnnoiseless + str0m), and
native-only unless noted. It bookends with a dev-tooling investment so the native UI
and (eventually) the voice path are reviewable without launching the app by hand.

## What shipped

**v0.485, Phase A - native mic loopback.** First proof the pure-Rust audio chain
works end to end: cpal (WASAPI) capture → unsafe-libopus encode/decode → playback,
glued by an `rtrb` lock-free ring. `src/net/voice.rs`.

**v0.486–0.487 - mic-test UX.** Toggle + animated RGB border + live input level
meter + input/output device selectors. Accepts **any** device sample format
(i16/u16/f32) and **any** sample rate via a streaming linear resampler to/from
48 kHz, so it isn't tied to one sound card's native format.

**v0.488 - the voice INPUT stack.** Mic gain 0–200% (clip-protected); noise FILTER
modes Off / Light (85 Hz biquad high-pass + noise gate) / Noise suppression (RNNoise
via the pure-Rust `nnnoiseless` crate - removes keyboards, coughs, fans even while
you're talking); TRANSMIT modes Open mic / Push-to-talk / Voice-activated /
Push-to-mute, with a bindable push key + a VAD threshold. All persisted to
`AppConfig` (new `VoiceFilterMode` + `VoiceTransmitMode` enums in `src/config.rs`).
6 DSP unit tests. UI in `src/gui/pages/settings.rs` `draw_audio_content`.

**v0.489, Phase B - bidirectional Opus MEDIA over str0m.** Added an audio m-line
(`add_media` + `Writer::write` + `Event::MediaData`) to the **existing** per-peer
`Rtc` in `src/net/webrtc.rs`, strictly **opt-in**: a connection only negotiates audio
when explicitly asked, so the P2P data mesh is completely unchanged. 2 SDP
negotiation tests.

**v0.490 - sane defaults.** Noise suppression (RNNoise) + Push-to-talk on CapsLock
out of the box. Push-key handling moved to **raw winit input** because egui has no
CapsLock concept and PTT has to work in-game, not just over a focused text field.
`src/config.rs` + `src/lib.rs`.

**v0.491, Phase C1 - relay JOIN registers for real.** A native voice-room join now
sends `{type:voice_room, action:join, room_id}` (the old `voice_join` was silently
ignored by the relay), so you actually appear in the channel's voice roster.

**v0.492, Phase C2 - WebRTC SIGNALING + connection.** Native speaks the web's
`voice_room_signal` protocol (signal `data` as a JSON object; newcomer-offers /
incumbents-wait glare rule; emits a `VoiceConnected` event). Interoperable with the
web client. `src/net/webrtc.rs` voice signaling path + `src/lib.rs` routing.

**v0.493 - voice is PER-CHANNEL (the model unification).** The voice room **is** the
text channel, keyed by the channel's own string id; the relay validates the channel's
`voice_enabled` flag rather than consulting a separate `voice_channels` table. Both
web and native now join by channel id. Relay touch:
`src/relay/handlers/{msg_handlers.rs,broadcast.rs}`, with `VoiceChannelData.id`
migrated `i64 → String`. **This is the fix for "clicking a channel's mic does
nothing":** voice rooms had been tied to a separate table keyed differently from the
channel, so a join targeted a room that didn't line up with the channel the user
clicked, and nothing happened. Collapsing the two onto the channel id made the mic
button do what it visually promised.

**v0.494–0.495, Phase D - LIVE voice audio.** Native captures mic → DSP →
Opus-encodes → sends to each connected peer; receives peers' Opus → **per-peer**
decode → mix → plays. native↔web is audible **both ways**. Shipped with the
**periodic-static fix**: play a peer's frames **in sequence via a per-peer queue**
rather than summing whatever arrived, and **pace the worker** instead of
busy-spinning. `src/net/voice.rs` `run_voice_session` + `src/lib.rs`. v0.495 also
folds in the dev tooling below.

## Dev tooling (no app/runtime behavior change)

- **Headless UI snapshots.** `src/gui/ui_snapshots.rs` renders native egui pages to
  PNGs via an **offscreen egui-wgpu + wgpu** pass - **no new dependency**.
  `egui_kittest` was the obvious choice but was rejected: an accesskit / egui-winit
  0.31.1 incompatibility. Output lands in `tests/snapshots/`, so the native UI can be
  reviewed without launching the binary.
- **New Justfile recipes.** `just verify` (both feature builds + lib tests + lints),
  `just lints` (the 4 `src/gui` file-scanner lints via standalone rustc, so they dodge
  the Windows LNK1318 PDB limit), `just snapshots` (render the UI PNGs), `just
  preflight` (untracked-source check + doc-link check + verify).

## Lessons

- **Every release restarts the live relay and force-reloads the web - which broke
  voice testing.** A relay restart clears the in-memory `voice_rooms` and drops all
  client WebSockets; the web client force-reloads when it sees `server_version` change
  (the `peer_list` handler in `app.js`). So a deploy mid-test kicks everyone out of
  voice. There are no production users yet (voice coordination happens over Discord),
  so full deploys are fine - but **batch them** rather than shipping one per tiny
  change while a voice session is live.
- **Opt-in negotiation kept the blast radius small.** Bolting audio onto the existing
  per-peer `Rtc` only when asked meant the entire data mesh and P2P paths stayed
  byte-for-byte unchanged through Phases B–D.
- **The static was an ordering bug, not a codec bug.** Summing concurrently-arrived
  frames and busy-spinning the worker produced periodic static; a per-peer sequential
  queue + a paced worker fixed it without touching encode/decode.
