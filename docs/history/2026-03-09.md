# 2026-03-09 — Game Engine Direction

Source: OpenClaw session with Shaostoul

## What Happened

Set strategic direction for the game layer. Target feel: No Man's Sky + Elite Dangerous + Star Citizen. Architecture decisions locked in.

## Architecture Decisions

- Custom engine runtime in Rust + wgpu
- Offline-first gameplay
- P2P primary with optional dedicated server mode
- Desync recovery prioritized to prevent hard session loss on brief disconnects
- SQLite snapshot/event backend (hybrid: snapshots + event log/audit)
- Start with Easy difficulty as default; L1/L2/L3 realism tiers for future

## Technical Work

Added and integrated core crates:
- core-lifeform-model, core-session-orchestrator, core-skill-progression
- core-teaching-graph, core-firstperson-controller, core-offline-loop
- persistence-sqlite, core-snapshot-sync

Added module crates with tests/scenarios:
- water, soil, crops, carpentry, electrical, plumbing, health-first-aid

Added CLI-playable harness (humanity-cli-game) with gameplay and DB commands. Added wgpu runtime shell (engine-wgpu-shell) — iterated from 2D prototype to true 3D mesh pipeline with depth, procedural terrain, and capsule stand-in character.

## User Feedback

- Flagged prototype as looking 2D/sidescroller — didn't match immersive expectations
- Requested 3D stand-in character on heightmapped terrain
- Requested 8px green diamond crosshair and better key/menu accessibility

## State at End of Session

- Offline no-combat milestone flow scriptable and passes objective progression in CLI
- Engine shell compiles with 3D terrain + capsule stand-in + third-person camera scaffold
